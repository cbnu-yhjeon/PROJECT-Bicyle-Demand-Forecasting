## 2. 스키마 기반 데이터 품질 체크포인트

### ✔ 날짜·시간 관련
- `rental_date`는 반드시 `datetime` 타입으로 변환 필요  
- `rental_hour`는 0~23 범위 밖 값 존재 여부 체크

### ✔ 범주형 표준화 필요
- `gender`: `M/F`로 통합  
- `rental_type`: 3종으로 확정(단체권/일일권/정기권)  
- `age_group`: 10/20/30 등의 정규화 가능

### ✔ 수치형 컬럼 검증
- `distance_m`, `usage_time_min`, `calories`는 음수가 나오면 절대 안 됨  
- 장거리/장시간 극단값(Outlier) 검사가 필요함

## 3. 데이터 활용 시 고려사항

### 🔎 모델링 시 유의할 점  
- 시간 단위 수요 예측 모델에 적합한 구조  
- 사용자 속성(성별·연령대·대여권 종류)을 포함 → 세그먼트 분석 가능  
- 연령대/성별 값은 반드시 정규화해야 모델 성능이 안정됨  

### 🧩 데이터 결합 측면  
- `station_id`를 기준으로 POI·위경도 데이터와 조인 가능  
- `rental_date`를 기준으로 기상데이터(ASOS)와 결합 가능
